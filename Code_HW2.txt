latency_measurement.js is a file that performs the latency measurement actions. The script defines the external urls of the vm instances located in Europe and the US. It uses performance.now from Node's perf_hooks to provide the timestamps needed to measure the request duration. The function avg_register generates a unique username then sends a JSON POST to ${url}/register (url is the external url of either Europe of the US) and ensures the request is fully completed. It records the elapsed time per request and then computes the average latency across 10 requests. The function avg_list performs 10 GET requests to ${url}/list, records the elapsed time per request, and returns the average similar to avg_register. In main, the script runs both avg_list and avg_register for both regions (Europe and US), then prints out the resulting averages.

consistency_measurement.js is a file that performs the observing eventual consistency actions. The script defines the external urls of the vm instances located in Europe and the US. It also sets the number of iterations (ITERS) to 100. The functions postRegister sends a JSON POST request to ${url}/register (url is the external url of either Europe of the US) and ensures the request is fully completed. The function getList performs a GET request to ${url}/list and attempts to parse the JSON response. If the JSON response is valid, the function returns the users array from the response. If it isn't valid, it returns an empty array. In main, the script runs a loop for 100 iterations. In each iteration, it generates a unique username and sends a POST request to the /register endpoint of the US instance using the function postRegister. Immediately after, it sends a GET request to the /list endpoint of the Europe instance using the function getList. Fianlly, it checks whether the newly registered username is present in the Europe list. If it is not found, the notFoundCount is incremented. After completing all iterations, the script prints the number of iterations and the number of times a newly registered username was not immediately visible in the other region.
